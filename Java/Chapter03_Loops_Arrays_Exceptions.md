# Chapter 03. 반복문과 배열 그리고 예외 처리

## 📅 2026-04-19 (for 문을 활용한 반복 합계 계산)

### ✅ 배운 내용

**for 문의 특징**

- 주로 반복 횟수가 명확히 정해져 있는 경우 사용함.
- 초기화식, 조건식, 반복 후 작업이 한 줄에 명시되어 있어 가독성이 좋습니다.

**초기문**  
for 문에서 초기문은 주로 조건식에서 사용하는 변수를 초기화함.

- 초기문은 시작할 때 한 번만 수행함.
- 콤마(,)로 분리하여 여러 문장을 나열할 수 있음.
- 초기문은 빈 상태로 두어도 되지만 끝에 세미콜론(;)은 있어야 함.

**조건식**  
매 루프 시작 전 검사하며, 조건식의 결과가 true이면 반복이 계속되며 false이면 for 문을 벗어남

```bash
for(초기문; true; 반복 후 작업) { //무한 반복
    ................
}
for(초기문; ; 반복 후 작업) { // 조건식이 비어 있으면 무한 반복
    ................
}
```

**반복 후 작업**

- 루프 몸체 실행 후 실행됨. 주로 루프 변수를 증감시키는 용도로 사용함.
- 반복 후 작업문에는 콤마(,)로 분리하여 여러 문장을 나열할 수 있음.

```bash
for(i = 0;, i < 10; i++, System.out.println(i)){
    ................
}
```

**반복문에 변수 선언**

- `for` 문의 초기화식에서 선언된 변수 i는 해당 for 문 블록 내부에서만 유효한 지역 변수임.
- 루프가 종료되면 해당 변수는 메모리에서 사라지므로 외부에서 접근할 수 없음.

```bash
for(int i = 0; i < 10; i++)
    System.out.print(i);
```

### ⚠️ Troubleshooting

- **문제**: `Syntax error on token ",", delete this token` 메시지 발생.
- **원인**: `for` 문의 소괄호 () 안에서 구문을 구분할 때 세미콜론(;) 대신 쉼표(,)를 사용함.
- **해결**: 문장의 끝을 의미하는 세미콜론을 사용하여 각 식을 명확히 구분함.

## 📅 2026-04-20 (while 문)

### ✅ 배운 내용

**while 문의 특징**

- **반복 횟수가 불분명할 때**: 사용자의 입력이나 특정 상태 변화에 따라 루프를 종료해야 하는 경우에 매우 적합합니다.
- **조건식 필수**: `for` 문과 달리, `while()` 소괄호 안에 조건식이 없으면 컴파일 오류가 발생함.
- **종료 조건의 중요성**: 루프 몸체 안에서 조건식의 결과가 언젠가 `false`가 되도록 설계해야 합니다. 그렇지 않으면 메모리 자원을 계속 점유하는 무한 루프에 빠지게 됩니다.

**데이터 타입 변환**

- 정수 데이터들의 평균을 구할 때 소수점까지 정확히 표현하기 위해 `(double)` 강제 형변환을 적용함.

## 📅 2026-04-21 (do-while 문)

### ✅ 배운 내용

**1. do-while 문의 특징**

- 선 실행, 후 검사: 조건식을 검사하기 전에 루프 몸체(do 블록)를 최소 한 번은 반드시 실행함.
- 세미콜론(;) 주의: `while (조건식);` 처럼 문장 끝에 반드시 세미콜론을 붙여야 컴파일 오류가 나지 않음.

**2. 문자의 산술 연산과 형변환**

- 자바에서 char 타입에 숫자를 더하면 결과가 int 타입으로 변함.
- 따라서 다시 문자로 출력하려면 (char)를 이용해 강제 형변환을 해줘야 함.

**3. while 문과의 차이점**
| 구문 | 조건 검사 시점 | 최소 실행 횟수 |
|:---:|:---:|:---:|
| while | 작업문 진입 전 (선 검사) | 0회 |
| do-while | 작업문 실행 후 (후 검사) | 1회 |

## 📅 2026-04-22 (중첩 반복문)

### ✅ 배운 내용

**1. 중첩 반복문의 이해**

- 반복문 안에 또 다른 반복문을 만들 수 있음.
- 외부 반복문이 한 번 실행될 때마다, 내부 반복문은 전체 루프를 다 돌아야 함.
  - 예: 구구단에서 바깥쪽(`i`)이 1단일 때, 안쪽(`j`)은 1부터 9까지 다 곱해진 뒤에야 다시 바깥쪽이 2단으로 넘어감.
- 반복은 몇 번이고 중첩 가능하지만, 너무 깊은 중첩 반복은 프로그램 구조를 복잡하게 함.

**2. 출력 형식 제어**

- 제어 문자 `\t` (Tab): 출력문의 정렬을 맞추기 위해 사용함.

- 줄바꿈 `System.out.println():` 내부 루프가 끝나는 시점에 호출하여, 단별로 행이 바뀌도록 제어함.

- **결과**: 구구단이 가로로 예쁘게 정렬되지 않고 한 줄씩 세로로 길게 출력됨.

## 📅 2026-04-23 (continue 문과 break 문)

### ✅ 배운 내용

반복문 내에 continue 문이나 break 문을 이용하면 반복의 흐름을 바꾸거나 반복에서 벗어나게 할 수 있음.

**continue 문**

- 반복문을 종료하지 않고 즉시 다음 반복으로 넘어가고자할 때 사용됨.
- for 문: continue 문을 만나면 '반복 후 작업'으로 분기함.
- while / do-while 문: 조건식을 검사하는 과정으로 분기함.

**break 문**

- break 문은 하나의 반복문만 벗어남.
- 중첩 반복문의 경우 안쪽 반복문에서 break가 실행되어도 바깥 쪽 반복문 실행은 유지됨.

## 📅 2026-04-24 (배열)

### ✅ 배운 내용

**배열이란?**  
배열은 같은 타입의 데이터들을 순차적으로 저장함.

**선언 및 생성**

- 배열에 대한 레퍼런스 변수 선언

```bash
int intAray[];
```

- 배열 선언 시 []에 크기를 지정하면 안 됨.

```bash
int intArray[5]; //컴파일 오류. 배열의 크기를 지정할 수 없음.
```

- 배열 생성

```bash
intArray = new int[5];
```

- 배열의 선언과 동시 생성
  다음과 같이 배열의 선언과 생성을 동시에 할 수 있음.

```bash
int intArray[] = new int[5];
```

**배열 인덱스와 배열 원소 접근**  
배열의 인덱스는 정수만 가능하며, 0부터 시작함.

**레퍼런스 치환과 배열 공유**

1. 레퍼런스란?  
   배열 이름은 배열 데이터를 저장하는 공간이 아니라, 실제 배열 데이터가 위치한 메모리 주소값을 '레퍼런스'라고 부름.

2. 레퍼런스 치환  
   한 배열 변수의 값을 다른 배열 변수에 대입하면, 배열의 복사본이 생기는 것이 아니라 주소값만 복사됨.

3. 배열 공유의 특징
   두 변수가 하나의 배열 데이터를 공유하게 되고,한쪽에서 배열 내용을 수정하면 다른 쪽도 수정됨.

## 📅 2026-04-25 (배열의 크기, length 필드)

### ✅ 배운 내용

**배열의 length 필드**

- 자바에서 배열은 객체로 다루며, 내부에 배열의 크기를 나타내는 `length` 필드가 존재함.
- 배열의 크기가 변경되어도 `length` 필드를 사용하면 프로그램에서 배열의 크기를 따로 관리할 필요가 없음.
- length 필드는 읽기 전용이라 값을 임의로 수정할 수 없음.

배열의 크기는 다음과 같이 간단히 알아낼 수 있음.

```java
int intArray [] = new int[5];
int size = intArray.length; //size는 5
```
