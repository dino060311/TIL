# Chapter 04. 클래스와 객체

## 📅 2026-05-02 (객체 지향과 자바)

### ✅ 배운 내용

**객체의 본질**

- 정의: 실세계의 모든 존재를 객체로 보며, 각 객체는 자신만의 고유한 상태(특성)와 행동을 가짐.
- 상호작용: 객체들은 서로 고립된 것이 아니라, 상대 객체에게 특정 행동을 요청하거나 정보를 주고받는 방식으로 상호 작용하면서 존재함.

**캡슐화 (Encapsulation)**

- 객체를 캡슐로 싸서 내부를 보호하고 볼 수 없게 하는 것으로 객체의 가장 본질적인 특징임.
- 자바에서의 구현: 클래스(Class)라는 캡슐을 사용함.

**상속 (Inheritance)**

- 개념: 슈퍼 클래스(부모)의 특성을 서브 클래스(자식)가 물려받는 것입니다.
- 슈퍼 클래스의 필드와 메소드를 물려받아 코드를 재사용함으로써, 코드 작성에 드는 시간과 비용을 줄임.

**다형성 (Polymorphism)**

- 개념: 같은 이름의 메소드가 클래스 혹은 객체에 따라 다르게 동작하도록 구현된 것을 말함.
- 같은 이름의 메소드가 클래스에 따라 다르게 동작하는 오버라이딩(Overriding) 등이 있습니다.

**객체 지향 언어의 목적**

- 소프트웨어의 재사용성 및 생산성 향상: 이미 만들어진 코드를 재사용하기 쉽고, 복잡한 시스템을 관리하기 용이하게 함으로써 개발 속도를 높임.
- 유지보수의 용이성: 수정이 필요한 부분만 고칠 수 있어 대규모 프로젝트 관리에 유리합니다.

**절차 지향 프로그래밍과 객체 지향 프로그래밍**

(1) 절차 지향 프로그래밍

- 중점: 실행하고자 하는 절차(순서)와 함수 중심임.
- 특징: 데이터를 처리하는 방법(알고리즘)을 순차적으로 나열하며, 데이터와 함수가 분리되어 있음.
- 단점: 프로그램 규모가 커지면 구조가 복잡해져 유지보수와 수정이 어려움.

(2) 객체 지향 프로그래밍

- 중점: 실세계의 개체를 모델링한 객체 중심임.
- 특징: 데이터(필드)와 그 데이터를 다루는 기능(메소드)을 하나의 객체로 묶어서 관리함.
- 장점: 코드의 재사용성이 높고 유지보수가 쉬우며, 대형 프로젝트에 적합함.
